# ☁️ Install Azure Site Recovery (ASR) Provider on Windows 11 Host

This guide explains how to set up your **Windows 11 machine (Hyper-V host)** to work with **Azure Site Recovery (ASR)**, so you can replicate on-prem Hyper-V VMs to Azure.

---

##  Prerequisites

- A **Recovery Services Vault** already created in Azure.
- **Hyper-V role** must be enabled on your Windows 11 host machine.
- At least one VM created in Hyper-V.

---

##  Step-by-Step: ASR Provider Setup

### 1. Go to Azure Portal  
- Navigate to your **Recovery Services Vault**.
- Under **Site Recovery Infrastructure**, click **+ Hyper-V Site**.

### 2. Add a Hyper-V Site
- Create a logical grouping for your Hyper-V hosts.  
  Example: `OnPremHyperVSite`

### 3. Add a Hyper-V Server
- Click **+ Hyper-V Server** under your new site.

### 4. Download the Required Files
- You’ll see two download links:
  - **Azure Site Recovery Provider Installer**
  - **Vault Registration Key (valid for 5 days)**

### 5. Install the ASR Provider
- On your **Windows 11 host**, run the ASR Provider installer.
- Accept license terms and proceed through the wizard.

### 6. Register the Host with Azure
- During installation, browse and select the **Vault Registration Key** file you downloaded.
- Let the setup complete. Your host is now registered with Azure.

---

##  Outcome

Your Windows 11 Hyper-V host is now connected to Azure Site Recovery. You can start replicating virtual machines to Azure for disaster recovery.
